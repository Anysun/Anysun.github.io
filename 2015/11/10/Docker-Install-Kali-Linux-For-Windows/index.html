Docker Install Kali Linux For Windows

在逛蘑菇哥的Blog发现上面链接的文章比较不错，想要尝试一翻，但可惜蘑菇哥在Macbook Pro下玩耍的。好吧，那我也在Windows下也学习学习。

以下引用蘑菇哥的文章：

什么是 Docker

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口（类似 iPhone 的 app）。几乎没有性能开销,可以很容易地在机器和数据中心中运行。最重要的是,他们不依赖于任何语言、框架包括系统。

为什么要使用 Docker

作为一种新兴的虚拟化方式，Docker 跟传统的虚拟化方式相比具有众多的优势。

首先，Docker 容器的启动可以在秒级实现，这相比传统的虚拟机方式要快得多。 其次，Docker 对系统资源的利用率很高，一台主机上可以同时运行数千个 Docker 容器。

容器除了运行其中应用外，基本不消耗额外的系统资源，使得应用的性能很高，同时系统的开销尽量小。传统虚拟机方式运行 10 个不同的应用就要起 10 个虚拟机，而Docker 只需要启动 10 个隔离的应用即可。

具体说来，Docker 在如下几个方面具有较大的优势：

  • 更快速的交付和部署
  • 更高效的虚拟化
  • 更轻松的迁移和扩展
  • 更简单的管理

看起来棒棒的，有木有，有木有。然后蘑菇哥开始在Mac OS下无尽的玩耍了，我也要开始在Windows下安装了。

Install Docker for Windows

由于Docker依赖于Linux特有的特性，你不能在Windows中本地运行。相反，你必须安装Docker Tools应用。这个应用会安装VirtualBox Virtual Machine(VirtualBox虚拟机)、Docker 和 Docker Toolbox管理工具。这3个小东西可以让你在Windows中运行Docker。

第一步：检查你的系统

1、你的主机必须运行在Windows 7.1，8/8.1或者更新的系统。

2、确保你的Windows系统支持硬件虚拟化技术，并且虚拟化是启用的。

Windows 8/8.1

在任务管理器中Performance选项卡中，在CPU下面，你可以看到：

如果虚拟化没有启用，你需要启用它。

Windows7

需要到微软官方网站下载Microsoft® Hardware-Assisted Virtualization Detection Tool 工具，来检查虚拟化是否启用，如果启用，你可以看到：

第二步：安装Docker Toolbox

在官网下载Docker Toolsbox：点击下载
安装没什么好说的了，Next Next Next Next….
安装成功后能找到3个图标：

在Docker中安装Kali

运行Docker Quickstart Terminal，然后拉取Kali的镜像，输入

$ docker pull kalilinux/kali-linux-docker

直到下载检查确认完毕。

查看镜像,输入

$ docker images

如图

可以查看到镜像ID，说明已经安装完毕，这时我们启动Kali系统，直接输入

$ docker run -i -t [IMAGE ID] /bin/bash

会启动失败，错误提示

cannot enable tty mode on non tty input

这个问题是MINGW64这些工具产生的，而不是docker本身产生的。而且这个问题还没有被关闭，所以等以后应该会有更好的解决方案吧。

这里有两种解决方案：
第一种在命令前面加上winpty

$ winpty docker run -it ubuntu /bin/bash

第二种方案更好

$ docker-machine ssh default

$ docker run -it ubuntu /bin/bash

这样可以正常运行了

Windows版本中Docker的优缺点

我在入门中新手上手体验的感觉

优点

  • 容器的启动速度确实很快
  • 镜像、容器管理很方便
  • 还有很多很多。。。

缺点

  • 新手不太容易上手，每次对容器的修改保存都需要commit等等
  • Docker需要依赖VirtualBox启动，总有种上当了的感觉。
  • 没有一丢丢图形界面，新手上手难度较大。
